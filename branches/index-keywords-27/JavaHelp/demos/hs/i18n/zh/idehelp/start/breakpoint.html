练习2：设置断点

设置断点可以帮助您跟踪程序中的出错点。在本练习中，您将在Blink.java的运行过程中设置一个断点。 程序运行到该断点位置就停止执行。

本教程所给出的源代码行号将假定您已经添加了编辑源代码中给出的附加代码。

  1. 选择调试->断点。
    将显示“Java Development Environment断点”窗口。
  2. 在“类名”框中输入Blink。
  3. 在“位置”框中输入45。
  4. 单击“设置断点”。
    源代码编辑器会高亮显示下面这一行： 
         if (x + w > d.width) {

    选中这一行，您就可以在程序循环执行时跟踪变量值的变化。大多数情况都会把断点设置在程序中 认为可能发生错误的那一行上。

Java Development Environment将一直运行这个程序，直到到达断点为止。这时，包含新断点的行就变成绿色， 表示程序将运行到这里停下来。“线程/堆栈”窗口也随即被更新。
