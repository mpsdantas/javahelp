제 4과: 메서드를 프로시저 단위로 실행하기

코드를 단계적으로 실행하는 것은 문제를 일으키는 라인을 찾아내는 또 하나의 방법입니다. 이 과에서는, 코드를 단계적으로 실행하면서 [스레드/스택] 창에서 변수 값의 변화를 추적하는 방법을 익힐 것입니다. 중단점을 없애는 방법도 익힐 것입니다.
  1. 편집/디버그 도구 모음에서 [프로시저 단위 실행] 단추 src="../../../../images/24/db-step-over_u.gif" align="BOTTOM">를 마우스로 누르십시오.
    Java Development Environment이 한 소스 라인을 실행하고 다음에서 멈춥니다:

    If (Math.random() < 0.5) {

    소스 편집기에서 해당 라인이 반전 표시되며, 중단점은 빨강색으로 표시되고 프로그램 실행이 멈춘 라인은 초록색으로 표시됩니다. 이 라인이 메서드 호출 라인인 경우, [프로시저 단위 실행] 명령의 경우 그 다음 라인까지 프로그램이 실행이 되고, [라인 단위 실행] 명령의 경우 해당 메서드의 첫번째 문장을 실행하고 멈춥니다.

    [스레드/스택] 창에 프로그램의 현재 상태와 프로그램의 데이터가 표시됩니다. 예를 들어, Blink.paint는 이제 라인 49에서 멈추었음을 표시합니다.

  2. [프로시저 단위 실행] 단추를 마우스로 두 번 더 누르고 디버거가 프로그램을 단계적으로 실행하는 방식을 관찰하십시오.
  3. 소스 편집기에서, 포인터를 라인 45로 옮기십시오. 라인 45에는 중단점이 있습니다.
  4. [중단점] 단추를 마우스로 누르십시오.
    중단점이 삭제됩니다.
