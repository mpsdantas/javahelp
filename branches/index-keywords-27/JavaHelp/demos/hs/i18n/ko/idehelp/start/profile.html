제 4과: 성능 분석 데이터 수집하기

이 과에서는 Java Development Environment 성능분석기를 사용하여 Blink 프로젝트를 실행하는 방법을 보여줍니다. 성능분석기는 프로그램의 성능을 분석하여 프로그램의 각 메서드에 얼마나 많은 시간이 사용되는지 그리고 각 메서드가 몇 번이나 실행되는지를 판단하는데 사용됩니다.
  1. 프로젝트 관리자에서 현재 프로젝트의 이름을 확인하십시오.
    현재 프로젝트가 Blink가 아니라면, Blink라는 프로젝트 이름을 마우스로 두 번 연속 누르십시오.
  2. Java Development Environment 주 도구 모음에서 [성능 분석] 단추를 마우스로 눌러서 Blink 프로젝트의 성능 분석을 시작하십시오.
    [애플릿 뷰어] 창이 화면에 나타납니다.
  3. 다음 단계로 넘어가기 전에 적어도 45 초를 기다리십시오.

    4. 이렇게 하면 쓸모있는 성능 분석 디스플레이를 만드는데 충분한 데이터가 마련됩니다.
  4. 애플릿 뷰어에서 [애플릿] -> [종료]를 선택하여 성능 분석 세션을 끝내십시오.
    프로그램을 성능 분석한 후에, [성능분석기] 창에 프로젝트 소스 디렉터리의 Blink.prof 파일에 기록된 데이터가 나타납니다.
  5. [성능분석기] 창에서, [필터] -> [없음]을 선택하여 시스템 클래스 데이터가 화면에 나타나게 하십시오.
  6. [보기] -> [이름별로 소트]를 선택하여 메서드 데이터가 영문자숫자 순으로 화면에 나타나게 하십시오.
  7. Blink.paint (Ljava/awt/Graphics;) 메서드를 선택하십시오.
  8. 성능 분석 데이터를 검토하십시오. 결과는 아래 그림과 비슷할 것입니다:

이 결과는 다음을 의미합니다:

  • Java Development Environment이 Blink.paint (Ljava/awt/Graphics;) 메서드를 호출하는데 93 밀리초를 사용했습니다.
  • Blink.paint (Ljava/awt/Graphics;)는 java.awt.Color.<init> (III), java.awt.Component.size (), 그리고 java.awt.Font.getSize () 등의 세 가지 메서드를 호출합니다.
  • Blink.paint (Ljava/awt/Graphics;) 메서드를 호출하는 두 가지 메서드는 java.awt.Component.dispatchEventImpl (Ljava/awt/AWTEvent;)와 java.awt.Component.update (Ljava/awt/Graphics;)입니다.
