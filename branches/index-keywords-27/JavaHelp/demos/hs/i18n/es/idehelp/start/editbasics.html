Conceptos importantes: atributos de proyecto

Los atributos definen las características individuales de un proyecto, como su nombre, tipo y la forma en que aparecen en una página HTML. El proyecto Blink que ha creado ha requerido atributos para determinar:
  • El tipo de proyecto.
  • El directorio que contiene los archivos fuente .java del proyecto.
  • El archivo de clase que contiene el método de applet init ()

Al crear el proyecto, proporcionó valores para estos atributos requeridos.

Blink también contiene numerosos atributos opcionales. Los atributos opcionales determinan cosas como la forma en que aparece el proyecto en el administrador de proyectos y las opciones de compilación utilizadas para construir el proyecto. Java Development Environment proporcionó valores predeterminados para estos atributos cuando creó el proyecto.

En la siguiente lección, editará los atributos que forman la etiqueta <applet>. Recuerde que cuando ejecutó el proyecto Blink en la lección anterior, Java Development Environment creó la etiqueta <applet> utilizando los requisitos mínimos para poner el applet en un página Web. Java Development Environment creó la etiqueta en el archivo Blink.tmp.html en el directorio Blink. El archivo actualmente tiene esta apariencia: 

<applet
   name="Blink"
   code="Blink.class"
   codebase="/su_directorio_inicial/jws/Blink"
   width="500"
   height="600"
   align="Top"
   alt=Si tuviera un examinador habilitado para Java, aquí vería una applet."
>     
<hr>Si el examinador reconociera la etiqueta de applet,
   aquí vería un applet.<hr>
</applet>

Blink.class es el archivo que contiene el código Java compilado de la aplicación. Codebase es el directorio que contiene el archivo Blink.class. La anchura y la altura definen el área, en pixels, que el examinador necesita para visualizar el applet.

No puede editar directamente el archivo .html. Java WorkShop actualiza el archivo basándose en las acciones realizadas utilizando el administrador de proyectos.
