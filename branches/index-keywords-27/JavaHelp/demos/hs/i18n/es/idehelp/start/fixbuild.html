Ejercicio 2: Corrección de errores en una construcción

Este ejercicio muestra cómo localizar rápidamente y corregir la línea de código fuente responsable de un error encontrado durante el proceso de construcción. Los errores de construcción normalmente son de sintaxis de lenguaje incorrecta, variables no declaradas y palabras clave mal escritas.

Si aún no ha añadido el código para hacer subir y bajar palabras en el proyecto Blink, hágalo ahora utilizando las instrucciones de Edición de código fuente.

Localización de errores

  1. Compruebe el nombre del proyecto actual en la ventana del administrador de proyectos.
    Si el proyecto actual no es Blink, pulse dos veces en en el nombre del proyecto Blink.
  2. Pulse en el botón "Construir" en la barra de herramientas principal de Java Development Environment.
    Java Development Environment compila el proyecto Blink utilizando el compilador de Java Development Environment predeterminado; este compilador es más rápido que el compilador de Java estándar de JDK.

    La pestaña "Construir" lista dos errores. El editor de código fuente resalta en amarillo la línea de código fuente responsable del primer error.

Corrección de errores

  1. Corrija el primer error, en la línea 55, añadiendo paréntesis alrededor de offset.
    La línea debe aparecer como sigue:
    if (offset)
  2. En la pestaña "Construir", seleccione el segundo error de construcción Blink.java: 61
    El editor de código fuente resalta la línea de código fuente responsable del siguiente error.
  3. Corrija el error añadiendo ; después de y = y / 2 en la línea 60.
    La línea debe aparecer como sigue:
    y = y / 2;
  4. Pulse en el botón "Guardar archivo" en la barra de herramientas de edición/depuración.
    Ahora el programa debe estar corregido y puede volverlo a construir.

Reconstrucción del proyecto

  1. Pulse en el botón "Construir" en la barra de herramientas principal.
    No debería recibir ningún mensaje de error. Si recibe alguno, regrese al editor de código fuente y vuelva a comprobar el código para ver si la información está bien introducida.
  2. Pulse en el botón "Ejecutar" en la barra de herramientas principal de Java WorkShop.
    Java Development Environment ejecuta el proyecto con el texto saltando de arriba a abajo, además de aparecer de forma intermitente.
