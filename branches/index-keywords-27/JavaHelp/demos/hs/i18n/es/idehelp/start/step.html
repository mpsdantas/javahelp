Ejercicio 4: Análisis de un método

El análisis del código es otra forma de llegar a la línea que está causando un problema. En este ejercicio aprenderá a realizar el seguimiento del código y detectar los cambios de los valores de las variables en la ventana "Subprocesos/Pila". También aprenderá a eliminar un punto de interrupción.
  1. Pulse en el botón "Analizar por procedimiento" en la barra de herramientas de edición/depuración.
    Java Development Environment ejecuta una línea de código fuente y para en:

    If (Math.random() < 0.5) {

    El editor de código fuente resalta en rojo la línea con un punto de interrupción y en verde la línea donde se paró la ejecución. Si la línea hubiera sido una llamada de método, el comando "Análisis por procedimiento" se habría ejecutado hasta la siguiente línea del programa. Si la línea hubiera sido una llamada de método y el usuario hubiera emitido el comando "Análisis línea a línea", Java Development Environment habría ejecutado la primera sentencia del método y habría parado.

    La ventana subprocesos/Pila muestra el estado actual del programa y sus datos. Por ejemplo, Blink.paint ahora indica que se para en la línea 49.

  2. Pulse en el botón "Análisis por procedimiento" dos veces más y observe cómo el depurador realiza el análisis línea por línea del programa.
  3. En el editor de código fuente, mueva el puntero hasta la línea 45, que era un punto de interrupción.
  4. Pulse en el botón "Punto de interrupción" .
    Java Development Environment suprime el punto de interrupción.
