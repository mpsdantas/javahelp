Java Development Environment-Werkzeuge

Die Java Development Environment-Werkzeuge werden durch Schaltflächen in der Java Development Environment-Hauptsymbolleiste (unterhalb der Menüleiste) dargestellt.

Folgende Schaltflächen stehen in der Hauptsymbolleiste zur Verfügung:

Erstellen/Kompilieren
Baut das aktuelle Projekt auf. Enthält das Projekt Fehler, werden die Fehler und Verknüpfungen mit Hilfe der Registerkarte "Erstellen" an den Quellcode übermittelt.
Projekt ausführen
Führt das aktuelle Projekt aus. Wenn es sich bei dem Projekt um ein Applet handelt, wird eine HTML-Datei erstellt und im Applet-Viewer ausgeführt. Handelt es sich bei dem Projekt um ein Standalone-Programmprojekt wird die Ausgabe an die Konsole gesendet. Ein Java-Paket oder eine Bean können nicht ausgeführt werden.
Ausführen und Profildaten sammeln
Führt den Profiler aus. Erstellt einen Bericht, in dem aufgeführt ist, wieviel Zeit für jede Programmethode aufgewendet und wie häufig die einzelnen Programmethoden ausgeführt werden.
Zeichenfolge suchen
Sucht Verweise auf Textzeichenfolgen in angegebenen Quelldateien.
Projekt-Manager
Erstellt, importiert, bearbeitet und entfernt Projekte.

Projekte stellen die Eckpfeiler von Java WorkShop dar. In einem Projekt werden die verschiedenen Dateien verfolgt, die zum Aufbauen, Durchsuchen und Ausführen eines Projektes in Java Development Environment sowie für die Fehlersuche in einem solchen Projekt benötigt werden. Projekte werden in Portfolios in Gruppen zusammengefaßt. Die folgenden Projekttypen stehen in Java Development Environment zur Verfügung:

  • Applet. Ein in der Java-Sprache geschriebenes Projekt, das in einem Browser ausgeführt wird, der Java unterstützt (z. B. HotJava).
  • Standalone. Ein in der Java-Sprache geschriebenes Programm, das außerhalb eines Web-Browsers ausgeführt wird (normalerweise aus einem MS-DOS-Befehlsfenster oder einer UNIX-Shell in der Betriebsumgebung UNIX).
  • Paket. Eine Gruppe von Klassen, die mit Hilfe des gleichen Paketnamens deklariert werden.
  • Bean. Wiederverwendbare Software-Komponenten, die im GUI-Builder bearbeitet werden können.
Attributeditor für GUI-Komponenten
Bearbeitet Eigenschaften von Schnittstellenkomponenten in GUI-Builder-Projekten.
Hauptsymbolleisten-Hilfe
Stellt Informationen über Java WorkShop bereit. Zum Anzeigen der Hilfe stehen verschiedene Möglichkeiten zur Verfügung:
  • Klicken Sie auf die Hilfeschaltfläche (das Fragezeichen) in der Java Development Environment-Hauptsymbolleiste, um Informationen über die Schaltflächen der Hauptsymbolleiste anzuzeigen.
  • Klicken Sie auf die Hilfeschaltfläche (das Fragezeichen) in der Symbolleiste "Bearbeiten/Debug", um Informationen über die Schaltflächen der Symbolleiste "Bearbeiten/Debug" anzuzeigen.
  • Wählen Sie im Menü "Hilfe" ein Element aus, für das Sie Hilfe anzeigen lassen möchten.
  • Klicken Sie in einem Dialogfeld oder einer Registerkarte auf die Schaltfläche "Hilfe".

Wenn Sie nicht sicher sind, welche Aktion durch eine Schaltfläche ausgeführt wird, können Sie eine Beschreibung der Aktion anzeigen lassen, indem Sie den Zeiger über der Schaltfläche positionieren. Die Beschreibung wird in der Fußzeile des Browsers angezeigt.
