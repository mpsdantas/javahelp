Viktiga begrepp: Projekt och Portfolios

Projekten är de grundläggande byggstenarna i Java Development Environment. Ett projekt organiserar de filer som krävs för att producera ett miniprogram, ett fristående program, ett Javapaket eller en böna. Ett projekt innefattar också information om sig självt, t.ex. projektets namn och huvudklassfil.

Java Development Environment lagrar projektinformation i en fil som benämns projektnamn.prj. Detta projekt lagrar källfilerna. Java Development Environment skapar filen genom att använda projektnamnet, källfilens namn och övrig information du ger när du skapar projektet.

När du har skapat projektet kan du lägga till och uppdatera dess information. Du kan t.ex. lägga till information om hur projektet ska visas på en HTML-sida, eller vilka kompileringsalternativ som ska användas vid uppbyggnaden av projektet. Du redigerar aldrig direkt i filen .prj. Java WorkShop uppdaterar filen baserat på de åtgärder du utför via Projekthanteraren.

Java Development Environment grupperar projekten tillsammans i en portfolio, på samma sätt som filer grupperas i kataloger. Oftast har varje användare en personlig portfolio, som bara innehåller de projekt som tillhör henne/honom. När du startar Java Development Environment för första gången befinner du dig "inuti" din personliga portfolio-den portfolio som du är "inuti" kallas för din aktuella portfolio.

Projektet som visas i källredigeraren kallas aktuellt projekt. Det aktuella projektet ställer in kontexten för alla verktyg i Java Development Environment. När du t.ex. använder felsökningsfunktioner felsöks aktuellt projekt.

Java Development Environment lagrar en portfolio i en fil som kallas portfolionamn.psf. Portfoliofilen listar sökvägsnamnet för varje .prj-fil, relativt portfolion.
