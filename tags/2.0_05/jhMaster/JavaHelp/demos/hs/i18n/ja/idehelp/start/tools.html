Java WorkShop のツール

メインツールバー (メニューのすぐ下にあります) 上にあるボタンは、Java WorkShop のツールを表わしています。

メインツールバーには、以下のボタンがあります。

「構築」
現在のプロジェクトを構築します。プロジェクトにエラーがある場合は、エラーメッセージと該当するソースコードへのリンクが「構築」タブ付きページに表示されます。
「実行」
現在のプロジェクトを実行します。プロジェクトがアプレットの場合は、HTML ファイルが作成されてアプレットビューアで実行されます。プロジェクトがスタンドアロンプログラムの場合は、コンソールに出力が表示されます。Java パッケージおよび JavaBeans は、このツールバーボタンを使用して実行することはできません。
「プロファイル」
各プログラム中のメソッドで費やされる時間や、各メソッドを実行する回数を確認することができます。
「検索」
指定した文字列をソースファイル中から検索します。
「プロジェクト」
プロジェクトマネージャを使用して、プロジェクトを作成、追加、編集、削除することができます。

プロジェクトは、Java WorkShop における基本単位です。Java WorkShop でプロジェクトを構築、ブラウズ、実行、デバッグするのに必要な各種のファイルは、プロジェクトごとに管理されます。複数のプロジェクトをまとめてグループ化したものは、ポートフォリオといいます。Java WorkShop では、以下の種類のプロジェクトを扱うことができます。

  • アプレット -- Java 言語で記述されたプログラムで、HotJava などの Java をサポートするブラウザで実行されます。
  • スタンドアロンプログラム -- Java 言語で記述されたプログラムで、Web ブラウザの外で実行されます。 通常、Microsoft Windows 95 および Microsoft Windows NT の環境では MS-DOS コマンドウィンドウから、UNIX オペレーティング環境ではシェルツールから実行します。
  • Java パッケージ -- 同じパッケージ名で宣言された Java クラスの集まりです。
  • JavaBeans -- GUI ビルダーで繰り返し利用することができるソフトウェアコンポーネントです。
「属性」
GUI ビルダーで作成しているプロジェクトのコンポーネント属性を設定することができます。
「ヘルプ」
Java WorkShop に関して説明されているオンラインヘルプを参照することができます。オンラインヘルプは、以下の方法で表示することができます。
  • メインツールバー上の「ヘルプ」ボタン (？ 記号) をクリックすると、メインツールバー上のボタンに関する情報が表示されます。
  • 編集デバッグツールバー上の「ヘルプ」ボタン (？ 記号) をクリックすると、編集デバッグツールバー上のボタンに関する情報が表示されます。
  • 「ヘルプ」メニューから、参照したい項目を選択します。
  • ダイアログまたはタブ付きページで、「ヘルプ」ボタンをクリックします。

また、ツールバー上のボタンにポインタを置くと、そのボタンによって実行される機能の簡単な説明が、ウィンドウのフッター (ウィンドウの左下隅) に表示されます。
