重要な概念: プロジェクトとポートフォリオ

プロジェクトは、Java WorkShop が扱う基本単位です。アプレット、スタンドアロンプログラム (main メソッドを持ち、アプレットビューアなしで実行可能なプログラムのこと)、 Java パッケージ、JavaBeans を作成するために必要なファイルを、プロジェクト単位でまとめて管理します。またプロジェクトには、プロジェクト名やメインクラスファイル名などのプロジェクトに関する情報も含まれています。

プロジェクトに関する情報は、「プロジェクト名.prj」というファイルに保存され、ソースファイルと同じディレクトリに置かれます。このファイルは、プロジェクト名やソースファイル名など、プロジェクト作成時に指定する情報を使用して作成されます。

プロジェクト作成後に、プロジェクトの情報を追加したり変更したりすることができます。たとえば、HTML ページにプロジェクトを表示する方法や、プロジェクトの構築に使用するコンパイラオプションなどの情報を追加することができます。.prj ファイルは、直接編集しないでください。このファイルは、プロジェクトマネージャを使用して設定した内容にもとづいて、Java WorkShop によって更新されます。

ファイルがディレクトリ単位でグループ化されるのと同じように、プロジェクトはポートフォリオという単位でグループ化されます。通常各ユーザーは、ユーザー個人に属するプロジェクトだけで構成される personal という名前の個人用ポートフォリオを持っています。Java WorkShop を最初に起動した時には、personal ポートフォリオが現在のポートフォリオになります。現在自分がいるポートフォリオのことを、「現在のポートフォリオ」といいます。

ソースエディタに表示されているプロジェクトは、「現在のプロジェクト」です。Java WorkShop のすべてのツールは、現在のプロジェクトを対象として動作します。たとえば、デバッガを使用している時は現在のプロジェクトがデバッグ対象となります。

ポートフォリオは、「ポートフォリオ名.psf」というファイルに保存されます。このファイルには、ポートフォリオから各 .prj ファイルへの相対パス名のリストが記録されています。
