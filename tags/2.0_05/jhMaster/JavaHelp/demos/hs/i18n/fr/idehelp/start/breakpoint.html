Exercice 2 : Définition d'un point d'arrêt

Il est très utile d'installer un point d'arrêt pour repérer les endroits à problème dans votre programme. Dans cet exercice, vous installerez un point d'arrêt dans Blink.java durant l'exécution du programme. Le programme s'éxecute jusqu'au point d'arrêt et stoppe.

Les numéros de ligne de source donnés dans ce didacticiel prennent en compte les lignes de code supplémentaires spécifiées dans Edition du code source.

  1. Choisissez Déboguer -> Points d'arrêt.
    La fenêtre Points d'arrêt de Java Development Environment s'affiche.
  2. Tapez Blink dans la zone de nom Classe.
  3. Tapez 45 dans la zone Emplacement.
  4. Cliquez sur Définir point d'arrêt.
    L'éditeur de source met la ligne en évidence : 
         if (x + w > d.width) {

    Cette ligne a été choisie afin que vous puissiez repérer le changement de valeur des variables pendant que le programme exécute la boucle. Dans la plupart des cas, vous installerez un point d'arrêt sur une ligne où vous pensez qu'il y a une erreur.

Java Development Environment fait tourner le programme jusqu'au point d'arrêt. La ligne contenant le nouveau point d'arrêt passe en vert pour indiquer où s'est arrêté le programme. La fenêtre Threads/Pile est mise à jour.
