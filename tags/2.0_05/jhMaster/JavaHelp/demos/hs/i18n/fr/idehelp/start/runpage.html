Exercice 2 : Mise à jour de la page d'exécution

Dans cet exercice, vous exécuterez votre fichier HTML nouvellement créé, et non le fichier par défaut Blink.tmp.html.
  1. Vérifiez dans le gestionnaire de projet le nom du projet en cours.
    Si le projet en cours n'est pas Blink, double-cliquez sur le nom de projet Blink.
  2. Choisissez Projet -> Editer.
    La fenêtre Edition de projet s'affiche.
  3. Cliquez sur l'onglet Exécution.
  4. Cliquez sur Oui pour URL de page d'exécution existant.
    Tous les champs de la fenêtre deviennent grisés, à l'exception de URL de page d'exécution existant. Lorsque ce dernier est paramétré sur Oui, Java WorkShop ne tient pas compte des informations figurant dans l'onglet Exécution, lorsque le projet est exécuté.
  5. Dans la zone URL de page d'exécution existant, saisissez le nouveau nom que vous venez de donner à votre nouveau fichier HTML mis à jour.
    Il n'est pas nécessaire de saisir le chemin d'accès complet. Java Development Environment ira chercher le fichier dans le répertoire source.
  6. Cliquez sur OK.
  7. Cliquez sur le bouton Exécuter de la barre d'outils principale.
    Java Development Environment charge le nouveau fichier HTML dans l'explorateur. Vous constaterez que le projet Blink s'exécute plus rapidement.
