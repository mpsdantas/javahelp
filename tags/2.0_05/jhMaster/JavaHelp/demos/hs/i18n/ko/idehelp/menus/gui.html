[GUI] 메뉴

[GUI Builder 보기] 이 메뉴 명령을 선택하면 Java Development Environment 창에 GUI 빌더 조절 아이콘이 추가됩니다. 이 메뉴 항목은 Visual Java GUI 빌더를 사용하여 만든 프로젝트에서만 활성 상태가 됩니다.
[GUI 파일 병합] 이 메뉴 명령을 선택하면 파일 선택창이 열리고 선택된 파일 (응용 프로그램에 포함시키려는 GUI 빌더 창이 들어 있는 파일)을 불러옵니다. 이 파일에는 반드시 .gui 확장자가 붙어 있어야 합니다.
[GUI 파일 저장 및 생성] 이 메뉴 명령을 선택하면 디스크의 project-name.gui 파일에 레이아웃이 저장되고 그 레이아웃에서 Java 소스 코드가 생성됩니다. 더 자세한 내용은 Java 소스 코드 생성하기를 참조하십시오.
[컴포넌트 잘라내기] 이 메뉴 명령을 선택하면 선택된 컴포넌트나 창을 잘라내어 클립보드에 올려 놓습니다. 잘라낸 컴포넌트나 창을 레이아웃의 다른 셀에 붙여 넣을 수 있습니다.
[컴포넌트 복사] 이 메뉴 명령을 선택하면 선택된 컴포넌트나 창을 복사하여 클립보드에 올려 놓습니다. 복사된 컴포넌트나 창을 클립보드에서 레이아웃의 다른 셀에 붙여 넣을 수 있습니다.
[컴포넌트 붙여넣기] 이 메뉴 명령을 선택하면 잘라내거나 복사한 컴포넌트나 창을 레이아웃의 다른 셀로 붙여 넣습니다.
[컴포넌트 삭제] 이 메뉴 명령을 선택하면 선택된 컴포넌트나 창을 지웁니다. 이 메뉴 명령은 클립보드에 복사본을 올려 놓지 않으므로 실행을 취소할 수 없습니다.
[속성 편집] 이 메뉴 명령을 선택하면 선택된 컴포넌트나 창의 속성이 나타나는 속성 편집기가 열립니다. 컴포넌트의 속성을 수정하는 방법에 관한 자세한 내용은 속성 편집기 도움말을 참조하십시오. 이 명령도 주 도구 모음에 해당 단추가 있습니다.
[열 추가] 이 명령을 선택하면 선택된 셀이나 컴포넌트의 왼쪽이나 오른쪽에 열을 추가합니다.
[행 추가] 이 명령을 선택하면 선택된 셀이나 컴포넌트의 위나 아래에 행을 추가합니다.
[새 창 추가] 이 메뉴 명령을 선택하면 [새 창] 대화 상자가 열립니다. 목록에서 특정한 컨테이너를 선택한 다음 [만들기]를 마우스로 눌러서 지정된 컨테이너를 위한 새 그리드백 레이아웃을 여십시오. 이 명령은 컴포넌트 팔레트의 왼쪽에 그에 해당하는 단추가 있습니다.
[창 유형 설정] 이 메뉴 명령을 선택하면 [주 컨테이너] 대화 상자가 열립니다. 주 컨테이너의 유형을 프레임이나 패널로 변경하려면 이 대화 상자를 사용하십시오.
[중첩 패널] 이 명령을 선택하면 선택된 레이아웃 패널 컴포넌트가 그리드백 레이아웃 패널 컴포넌트 안에 (제자리에) 중첩됩니다.
[팔레트 사용자 정의] 이 메뉴 명령을 선택하면 팔레트 편집기가 열립니다. 팔레트 편집기를 사용하면 팔레트 폴더 안에 있는 컴포넌트들을 수정하거나 삭제하거나 재배열할 수 있습니다.
[팔레트로 컴포넌트 임포트하기] 이 메뉴 명령을 선택하면 [팔레트로 컴포넌트 임포트하기] 대화 상자가 열립니다. 이 대화 상자를 사용하면 JavaBeans이나 shadows 클래스가 있는 컴포넌트를 임포트할 수 있습니다. 더 자세한 내용은 컴포넌트 임포트하기를 참조하십시오.
[팩토리 팔레트로 재설정] 이 메뉴 명령을 선택하면 기본적인 GUI 빌더 팔레트의 내용 ([AWT] 탭, [패널] 탭, 및 [추가] 탭)이 원래의 상태로 돌아갑니다.
[선택한 컴포넌트의 Bean 정보] 이 메뉴 명령을 선택하면 프로퍼티, 이벤트, 메서드 등을 포함하여 빈 관련 정보가 나오는 [Bean 정보] 대화 상자가 열립니다.
[디자인에서 컴포넌트 클래스 업데이트] 이 메뉴 명령을 선택하면 Bean 클래스 목록이 나오는 [디자인에서 컴포넌트 클래스 업데이트] 대화 상자가 열립니다. 갱신할 클래스(또는 클래스들)을 선택하십시오. 팔레트의 컴포넌트들은 갱신이 되지만, [레이아웃] 창의 컴포넌트들은 갱신이 되지 않습니다. 재로딩이 필요하면 Reload Component in Design 대화 상자가 나타납니다.
[클래스 사용 보고서] 이 메뉴 명령을 선택하면 현재 프로젝트에서 사용된 컴포넌트 클래스 목록이 화면에 나타납니다.
