Übung 2: Festlegen eines Haltepunkts

Das Festlegen eines Haltepunkts ist hilfreich für die Ermittlung von Schwachstellen im Programm. In dieser Übung werden Sie in Blink.java während der Ausführung des Programms einen Haltepunkt festlegen. Das Programm wird bis zum Haltepunkt ausgeführt und dann angehalten.

Bei den in diesem Trainingshandbuch angegebenen Programmzeilenzahlen wird davon ausgegangen, daß Sie den im Kapitel Bearbeiten von Quellcode behandelten zusätzlichen Code hinzugefügt haben.

  1. Wählen Sie "Fehlersuche" -> "Haltepunkte".
    Das Fenster "Java Development Environment Haltepunkte wird angezeigt.
  2. Geben Sie Blink in das Feld "Klassenname" ein.
  3. Geben Sie 45 in das Feld "Position" ein.
  4. Klicken Sie auf "Haltepunkt einstellen".
    Der Quelleditor hebt die folgende Zeile hervor:
         if (x + w > d.width) {

    Diese Zeile wurde ausgewählt, so daß Sie die Änderungen der Variablenwerte wiederfinden können, wenn das Programm in der Schleife ausgeführt wird. In den meisten Fällen wird ein Haltepunkt in der Zeile festgelegt, in der Sie das Auftauchen eines Programmfehlers vermuten.

Java Development Environment führt das Programm aus, bis der Haltepunkt erreicht ist. Die den neuen Haltepunkt enthaltende Zeile wird grün, um das Anhalten der Programmausführung anzuzeigen. Das Fenster"Threads/Stapel wird aktualisiert.
