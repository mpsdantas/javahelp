중요한 개념: 프로젝트와 포트폴리오

프로젝트는 Java Development Environment의 기본 빌드 구성 단위입니다. 프로젝트는 애플릿, 독립형 프로그램, Java 패키지, Bean 등을 생성하는데 필요한 파일들을 체계적으로 조직합니다. 프로젝트에는 또한 프로젝트의 이름과 main 클래스 파일과 같은 프로젝트 자체에 대한 정보도 들어 있습니다.

Java Development Environment은 projectname.prj라고 하는 파일에 프로젝트 정보를 저장합니다. 이 프로젝트는 소스 파일을 저장합니다. Java Development Environment은 프로젝트 이름, 소스 파일 이름, 및 기타 프로젝트를 만들 때 지정하는 정보를 사용하여 이 파일을 만듭니다.

프로젝트를 만든 후에, 그 정보를 추가하거나 수정할 수 있습니다. 예를 들어, HTML 페이지에 프로젝트를 나타나게 하는 방법이나 프로젝트를 빌드하는데 어느 컴파일러 옵션을 사용할 것인가에 관한 정보를 추가할 수 있습니다. 절대로 .prj 파일을 직접 수정하지 마십시오. 프로젝트 관리자를 사용하여 사용자가 수행하는 동작을 기초로 파일이 자동으로 갱신됩니다.

파일을 그룹으로 묶어서 디렉터리에 넣는 것과 같은 방식으로 Java Development Environment은 프로젝트를 함께 묶어서 포트폴리오에 넣습니다. 일반적으로 각 사용자는 자기에게 속한 프로젝트만을 담고 있는 personal 포트폴리오를 가지고 있습니다. Java Development Environment을 처음 실행하면, 이 personal 포트폴리오 "안에" 들어갈 것입니다 - 사용자가 "안에" 들어간 포트폴리오를 현재 프로젝트라고 합니다.

소스 편집기에 나타난 프로젝트를 현재 프로젝트라고 합니다. 현재 프로젝트는 Java Development Environment에서 사용하는 모든 도구의 환경이 됩니다. 예를 들어, 디버깅 도구를 사용하면 현재 프로젝트를 대상으로 실행이 됩니다.

Java Development Environment은 portfolioname.psf라고 하는 파일에 포트폴리오를 저장합니다. 포트폴리오 파일에는 그 포트폴리오와 관련이 있는 .prj 파일의 경로 이름이 나옵니다.
