제 2과: 빌드 중에 오류 바로잡기

이 과에서는 빌드 중에 발견된 오류의 원인이 되는 소스 라인을 빠르게 찾아서 바로잡는 방법을 보여 줍니다. 빌드 오류에는 보통, 부정확한 언어 구문, 선언되지 않은 변수, 그리고 철자가 잘못된 키워드 등이 포함됩니다.

Blink 프로젝트에 단어들을 튀어오르게 하는 코드를 아직 추가하지 않았다면, 소스 코드 수정하기를 참조하여 지금 코드를 추가하십시오.

오류 찾아내기

  1. 프로젝트 관리자에서 현재 프로젝트의 이름을 확인하십시오.
    현재 프로젝트가 Blink가 아니라면, Blink라는 프로젝트 이름을 마우스로 두 번 연속 누르십시오.
  2. Java Development Environment 주 도구 모음에서 [빌드] 단추 를 마우스로 누르십시오.
    Java Development Environment은 기본 설정에 따라 Java Development Environment Compiler를 사용하여 Blink 프로젝트를 컴파일합니다. 이 컴파일러는 일반적인 JDK Java Compiler보다 빠릅니다.

    [빌드] 탭에는 두 가지 오류가 나옵니다. 소스 편집기는 첫번째 오류의 원인이 된 소스 라인을 노랑색으로 반전 표시합니다.

오류 바로잡기

  1. offset 앞뒤에 괄호를 추가하여 라인 55에 있는 첫번째 오류를 바로잡으십시오.
    코드 라인이 다음과 같은 모양이 되어야 합니다:
    if (offset)
  2. [빌드] 탭에서, 두 번째 오류인 Blink.java: 61를 선택하십시오.
    소스 편집기에서 그 다음 오류의 원인이 된 소스 라인이 반전 표시됩니다.
  3. 라인 60에서 y = y / 2 다음에 ;를 추가하여 오류를 바로잡으십시오.
    코드 라인이 다음과 같은 모양이 되어야 합니다:
    y = y / 2;
  4. 편집/디버그 도구 모음에서 [파일 저장] 단추를 마우스로 누르십시오.
    이제 프로그램의 오류가 바로잡아졌으므로 프로그램을 다시 빌드할 수 있습니다.

프로젝트를 다시 빌드하기

  1. 주 도구 모음에서 [빌드] 단추 를 마우스로 누르십시오.
    아무 오류 메시지도 나오지 않을 것입니다. 오류 메시지가 나오면, 소스 편집기로 돌아가 정보를 올바로 입력했는지 소스 코드를 다시 확인하십시오.
  2. Java Development Environment 주 도구 모음에서 [실행] 단추 를 마우스로 누르십시오.
    프로젝트가 실행되고 텍스트는 깜박거리면서 위 아래로 오르락 내리락합니다.
