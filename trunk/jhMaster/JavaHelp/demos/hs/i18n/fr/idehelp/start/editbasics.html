Concepts importants : Attributs de projet

Les attributs définissent les caractéristiques propres à un projet telles que son nom, son type, et son aspect sur une page HTML. Le projet Blink que vous avez créé nécessitait des attributs qui déterminaient :
  • Le type du projet
  • Le répertoire contenant les fichiers source .java du projet
  • Le fichier de classe contenant la méthode init () de l'appliquette.

Vous avez fourni la valeur de ces attributs au moment de la création du projet.

Blink contient également de nombreux attributs facultatifs. Ces derniers déterminent par exemple l'aspect du projet dans le gestionnaire de projet ainsi que les options de compilateur employées pour générer celui-ci. Java Development Environment a fourni la valeur par défaut de ces attributs lors de la création du projet.

Dans la leçon suivante, vous éditerez les attributs constituant la balise <applet>. N'oubliez pas que lorsque vous avez exécuté le projet Blink dans la leçon précédente, Java Development Environment avait créé la balise <applet> en utilisant les conditions minimales pour placer l'appliquette dans une page Web. Java Development Environment a créé cette balise dans le fichier Blink.tmp.html de votre répertoire Blink. Le fichier a maintenant l'aspect suivant : 

<applet
   name="Blink"
   code="Blink.class"
   codebase="/votre_rép_d'origine/jws/Blink"
   width="500"
   height="600"
   align="Top"
   alt="Si vous aviez un explorateur compatible Java, vous verriez une appliquette ici."
>
<hr>Si votre explorateur a reconnu la balise d'appliquette,
   vous voyez une appliquette ici.<hr>
</applet>

Blink.class est le fichier contenant le code Java compilé de l'appliquette. Codebase est le répertoire contenant le fichier Blink.class. La largeur et la hauteur définissent la zone, en pixels, dont l'explorateur a besoin pour afficher l'appliquette.

Vous ne pouvez pas éditer le fichier .html directement. Java Development Environment le mettra à jour en fonction des actions que vous effectuez dans le gestionnaire de projet.
