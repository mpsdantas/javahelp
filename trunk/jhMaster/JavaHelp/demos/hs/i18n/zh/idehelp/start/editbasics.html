重要概念：项目属性

属性定义了项目的独有特性，比如它的名字、类型及其在HTML页面中出现的方式。 您所创建的Blink项目 就包含一些要求的属性，它们定义了：
  • 项目类型
  • 包含项目的.java源文件的 目录
  • 包含applet init ()方法的类文件。

这些要求的属性值是在您创建该项目时提供的。

Blink还包括许多任选的属性。这些任选属性决定了诸如项目出现在项目管理器中的外观， 以及生成项目所使用的编译器选项等事项。这些属性的缺省值是在创建项目时由Java Development Environment提供的。

在下一课中，您将对组成<applet>标签的属性进行编辑。现在让我们回想在前一课运行Blink项目时， Java Development Environment使用最低要求创建了<applet>标签，从而将applet小程序放入web页面中。 Java Development Environment把该标签创建在Blink目录的Blink.tmp.html文件中。该文件当前大约是这样的： 

<applet
   name="Blink"
   code="Blink.class"
   codebase="/your_home_directory/jws/Blink"
   width="500"
   height="600"
   align="Top"
   alt="If you had a java-enabled browser, you would see an applet here."
>     
<hr>If your browser recognized the applet tag,
   you would see an applet here.<hr>
</applet>

Blink.class是包含applet小程序的已编译Java代码的文件。Codebase是包含Blink.class文件的目录。 宽度和高度定义了浏览器显示该applet小程序所需的区域，以象素为单位。

您不可以直接编辑.html文件。Java Development Environment将根据您使用项目管理器所执行的动作来更新这个文件。
