Viktiga begrepp: Projektattribut

Attribut definierar projektets individuella egenskaper som namn, typ och hur det visas på en HTML-sida. Blink-projektet som du skapade krävde attribut för:
  • Projekttyp
  • Katalogen som innehåller .java-källfilerna för projektet
  • Klassfilen som innehåller init ()-metoden för miniprogrammet.

När du skapade projektet angav du värden för dessa attribut.

Projektet Blink innehåller dessutom flera attribut som inte är obligatoriska. Dessa avgör till exempel hur projektet visas i projekthanteraren och vilka kompileraralternativ som används när projektet byggs. Java Development Environment tillhandahöll standardvärden för dessa attribut när du skapade projektet.

I nästa lektion kommer du att redigera attributen som utgör märket <miniprogram>. Tänk på att när du körde Blink-projektet i föregående lektion skapades märket <miniprogram> med endast de krav som behövs för att visa ett miniprogram på en webbsida. Java Development Environment skapade märket i filen Blink.tmp.html i katalogen Blink. Just nu ser filen ut ungefär så här: 

<applet
   name="Blink"
   code="Blink.class"
   codebase="/your_home_directory/jws/Blink"
   width="500"
   height="600"
   align="Top"
   alt="If you had a java-enabled browser, you would see an applet here."
>
<hr>Om läsaren kände igen märket miniprogram visas ett miniprogram här.<hr>
</applet>

Filen Blink.class innehåller miniprogrammets kompilerade Java-kod. Katalogen Kodbas innehåller filen Blink.class. Bredden och höjden på det område som webbläsaren behöver för att visa miniprogrammet anges i bildpunkter.

Du behöver inte redigera filen .html direkt, utan Java Development Environment kommer att uppdatera den utifrån åtgärderna som du utför i projekthanteraren.
