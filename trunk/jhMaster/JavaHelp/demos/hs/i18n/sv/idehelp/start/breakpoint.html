Övning 2: Ange brytpunkt

Att ange en brytpunkt är användbart för att kunna spåra felaktigheter i ditt program. I den här övningen kommer du att ange en brytpunkt i Blink.java när programmet körs. Programmet körs fram till brytpunkten och stannar där.

Det radantal i källan som anges i den här övningen tar för givet att du har lagt till den extra koden som ges i Redigera källkod.

  1. Välj Felsöka -> Brytpunkter.
    Java Development Environments fönster Brytpunkter visas.
  2. Skriv Blink i klassens namnruta.
  3. Skriv 45 i rutan Placering.
  4. Klicka på Ange brytpunkt.
    Källredigeraren markerar raden: 
         if (x + w > d.width) {

    Den här raden valdes så att du kan spåra ändringen i variablernas värde när programmet går igenom slingan. I de flesta fall kommer du att ange en brytpunkt på en rad där du tror att ett programfel uppstår i ditt program.

Java Development Environment kör programmet tills det når brytpunkten. Raden som innehåller den nya brytpunkten blir grönfärgad för att visa var någonstans programkörningen stannade. Fönstret Trådar/stack uppdateras.
