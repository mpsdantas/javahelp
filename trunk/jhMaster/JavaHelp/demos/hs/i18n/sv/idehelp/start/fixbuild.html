Övning 2: Åtgärda fel i ett bygge

Med den här övningen får lära dig hur du snabbt hittar den källrad som innehåller ett fel som påträffades under byggprocessen, och åtgärdar felet. Byggfel beror ofta på felaktig språksyntax, odeklarerade variabler och felstavade nyckelord.

Om du ännu inte har lagt till koden som får orden att studsa i projektet Blink, bör du göra det nu enligt instruktionerna i Redigera källkod.

Söka efter fel

  1. Kontrollera namnet på det aktuella projektet (det finns i projekthanteraren).
    Om det aktuella projektet inte är ett Blink-projekt dubbelklickar du på Blink-projektets namn.
  2. Klicka på knappen Bygg i verktygsfältet i Java Development Environments huvudfönster.
    Java Development Environment använder som standard Java-kompileraren för att kompilera Blink-projektet. Den är snabbare än standardkompileraren JDK Java.

    På fliken Bygg listas två fel. Källraden som innehåller det första felet markeras med gult.

Åtgärda fel

  1. Åtgärda det första felet på rad 55 genom att lägga till parenteser runt offset.
    Raden bör se ut så här:
    if (offset)
  2. Markera det andra felet, Blink.java: 61 på fliken Bygg.
    Källraden som innehåller felet markeras.
  3. Åtgärda felet genom att lägga till ett semikolon (;) efter y = y / 2 på rad 60.
    Raden bör se ut så här:
    y = y / 2;
  4. Klicka på knappen Spara fil i verktygsfältet för redigering och felsökning.
    Programmet bör nu vara åtgärdat så att du kan bygga om det.

Bygga om projektet

  1. Klicka på knappen Bygg i huvudverktygsfältet.
    Inget felmeddelande bör uppstå. Om det händer återgår du till källredigeraren och kontrollerar om du har skrivit rätt.
  2. Klicka på knappen Kör i verktygsfältet i Java Development Environments huvudfönster.
    När projektet körs både studsar och blinkar texten.
